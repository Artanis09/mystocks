## 2. 학습 알고리즘 상세 설명 (Algorithm Architecture)

### 2.1 CatBoost (Categorical Boosting) Classifier
본 시스템은 Gradient Boosting Decision Tree(GBDT)의 진화형인 **CatBoost**를 핵심 엔진으로 사용합니다.

*   **Ordered Boosting:** 시계열 데이터에서 발생하는 데이터 누수(Data Leakage)를 방지하기 위해 훈련 세트의 순서를 고려하는 부스팅 기법을 적용합니다.
*   **Symmetric Trees:** 대칭형 트리를 구축하여 예측 속도가 매우 빠르며, 복잡한 주식 데이터 내의 노이즈에 강한 내성을 가집니다.
*   **Multi-Class Objective:** `MultiClass` 손실 함수를 사용하여 다음 날의 수익률 구간을 7개 등급으로 분류합니다. 이는 이진 분류보다 훨씬 세밀한 '상승 에너지' 파악을 가능하게 합니다.

### 2.2 타겟 레이블링 (Target Binning Strategy)
다음 날 수익률($r_{t+1}$)을 기준으로 다음과 같이 7개 클래스를 정의하였습니다:
*   **Class 0:** $r_{t+1} < 2\%$ (중립 또는 하락)
*   **Class 1:** $2\% \le r_{t+1} < 5\%$ (소폭 상승)
*   **Class 2:** $5\% \le r_{t+1} < 8\%$ (강세)
*   **Class 3:** $8\% \le r_{t+1} < 12\%$ (급등)
*   **Class 4:** $12\% \le r_{t+1} < 19\%$ (대급등)
*   **Class 5:** $19\% \le r_{t+1} < 29\%$ (준 상한가)
*   **Class 6:** $r_{t+1} \ge 29\%$ (상한가)

## 6. 최종 실전 필터 알고리즘 (Final Trading Logic)

실전(운영) 매매 기준은 **Filter2 고정**입니다. (웹앱/백엔드 예측 API도 Filter2만 허용)

### Filter2 (운영 기준)

1.  **신뢰도 필터 (Probability >= 70%):** 모델의 분류 확률값 중 상위 등급(Class 1~6)의 합산 확률(`prob_up`)이 70% 이상인 종목만 후보로 선정.
2.  **유동성 필터 (Market Cap >= 500억):** 시가총액 500억 이상만 후보로 선정.
3.  **거래 활성화 필터 (Volume > 0):** 거래정지 종목을 원천 제외하기 위해 당일 거래량이 있는 종목만 선정.
4.  **리스크 필터 (Daily Strength):** 당일 시가 대비 종가 등락률(`close/open - 1`)이 -5% 미만인 종목 제외.
5.  **강한 음봉 제거 (return_1d >= -5%):** 전일 대비 수익률(`return_1d`)이 -5% 미만인 종목 제외.
6.  **상한가 락 근사 제거 (return_1d < 29.5%):** 상한가 근처 급등(현실적으로 체결이 어려운 구간)을 근사적으로 제거.
7.  **선정/정렬:** 기대수익률(`expected_return`) 내림차순으로 정렬 후 Top-5를 최종 추천.
8.  **매매 방식:** 선정된 Top-5 종목에 자산을 1/N로 균등 배분하여 당일 종가 매수/익일 종가 전액 매도.

### 7.1 데이터 환경 구축
본 시스템은 Python 3.10+ 환경에서 작동하며 필수 패키지는 다음과 같습니다.

직접 설치 시(최소): `pip install catboost lightgbm pandas numpy pyarrow matplotlib tqdm`

### 7.2 단계별 실행 프로세스
1.  **데이터 수집 (EOD: 자동 수집 스케줄러)**
        - **오후 4:00 (16:00):** 전 종목 EOD 데이터 자동 수집 (`workers 2`).
        - **오후 8:00 (20:00):** 익일 거래를 위한 유니버스 자동 구축.
        - 수동 명령: `python .\crawl.py --mode eod --workers 2 --merge`

2.  **유니버스 구축 로직**
        - **대상:** 전일 상한가(29.5% 이상 상승) 종목.
        - **필터:** 시가총액 조건 제거 (`MIN_MARKET_CAP = 0`).
        - **시간 제한:** 데이터 수집 시간인 오후 4시~6시 사이에는 유니버스 구축이 금지됨.
        - **초기화:** 유니버스 재구축 시 현재 보유 중이거나 청산 대기 중인 종목을 제외한 기존 포지션 데이터는 모두 삭제되어 데이터 섞임을 방지함.

3.  **전처리 및 학습:** `python -m ml.run_all --train-only`
4.  **성과 검증:** `python ml/backtest_marketcap_50b.py`
5.  **실전 추론:** `python -m ml.inference`

### 7.3 crawl.py Argument
`crawl.py`는 모드 기반으로 동작합니다.

- 도움말: `python .\crawl.py --help`
- 주요 옵션
    - `--mode eod|intraday`
        - `eod`: 전체 종목 수집 후 유니버스 캐시 생성
        - `intraday`: 현재는 자동 스케줄러에서 제외됨 (필요시 수동 실행)
    - `--target-date YYYY-MM-DD` (기본: 오늘)
        - 단일 날짜만 재수집/업데이트
    - `--start-date YYYY-MM-DD --end-date YYYY-MM-DD`
        - 날짜 범위를 지정하여 해당 기간을 하루씩 재수집/업데이트
        - 기본은 주말을 자동 skip (KRX 휴장일은 자동 판별 불가 → 데이터가 없으면 스킵)
    - `--include-weekends`
        - 날짜 범위 업데이트 시 주말도 시도
    - `--workers N` (자동 스케줄은 2 사용)
    - `--merge` (EOD에서 기존 파티션이 있을 때 code 단위 갱신)

## 8. 실시간 자동매매 상세 전략 (Real-time Auto-Trading)

상한가 종목 유니버스를 대상으로 한 시가 갭 매매 전략입니다.

### 8.1 진입 전략 (Entry Strategy)
1.  **갭 조건 필터링:** 장 시작 후 시가가 전일 종가 대비 **+2% 이상**인 종목만 '감시중' 상태로 유지. 갭 2% 미만인 종목은 즉시 **'탈락(DISQUALIFIED)'** 처리되어 당일 매매에서 제외됨.
2.  **주문 방식:** **시가 지정가(Limit Order at Open Price)** 주문. 급등에 따른 슬리피지를 방지하기 위해 당일 시가로 매수 주문을 제출함.
3.  **진입 구간:** 오전 09:00 ~ 09:30.
4.  **미체결 처리:** 오전 09:30까지 체결되지 않은 진입 주문은 전량 자동 취소됨.

### 8.2 청산 전략 (Exit Strategy)
1.  **목표 수익 (TP):** +10.0% (기본값).
2.  **전략 손절 (SL):** **고정 수익률(-3%) 손절을 제거**하고, **현재가가 전일 종가(Prev Close)를 하회할 경우** 즉시 시장가 매도하여 손실을 방지함.
3.  **장마감 청산 (EOD):** 오후 15:20 ~ 15:28 사이 보유 종목 전량 시장가 매도.

본 섹션은 실제 KIS API와 연동되어 구동되는 **[auto_trading_strategy1.py](auto_trading_strategy1.py)** 엔진의 핵심 매매 로직을 설명합니다. 기존 통계 모델 기반 예측에서 한 단계 나아가, 변동성 돌파와 눌림목 지지를 결합한 **강화된 운용 규칙**을 적용합니다.

### 9.1 유니버스 선정 (Universe Selection)
매일 저녁 8시(20:00), 다음의 조건을 모두 충족하는 당일 급등주를 익일 매매 대상 유니버스로 자동 편입합니다.

1.  **가격 제한폭:** 당일 종가 기준 수익률이 +29.5% 이상 (상한가 종목)
2.  **시가총액:** 1,000억 원 이상 (`MIN_MARKET_CAP = 1000`)
3.  **거래대금:** 당일 거래대금 300억 원 이상 (`MIN_TRADING_VALUE = 300`)

### 9.2 시장 필터 (Market Filter)
개별 종목의 상태와 관계없이, 전체 시장의 추세에 따라 진입 여부를 결정합니다.

*   **KOSPI 5일 이평선 필터:** KOSPI 지수가 5일 이동평균선 위에 있을 때만 신규 진입을 허용합니다. 하락장(역배열)에서의 변동성 손실을 최소화합니다.

### 9.3 진입 전략 (Entry Strategy) - "갭상승 후 눌림목 매수"
장 시작 후 즉시 매수하지 않고, 특정 가격 조정을 확인한 후 진입하여 안전 마진을 확보합니다.

1.  **갭 조건 확인 (09:00):** 시가(Open)가 전일 종가 대비 **+2% ~ +5%** 범위 내에서 시작해야 합니다.
    *   +2% 미만 갭: 상승 탄력 부족으로 판단하여 탈락 (`DISQUALIFIED`)
    *   +5% 초과 갭: 과열 및 추격 매수 위험으로 판단하여 탈락 (`DISQUALIFIED`)
2.  **가격 조정 확인:** 갭 조건 충족 후, 주가가 다시 내려와 **전일 종가(0% 지점)**에 도달할 때 매수 주문을 실행합니다.

### 9.4 청산 전략 (Exit Strategy)
수동 개입 없는 자동 청산을 통해 원칙 매매를 고수합니다.

1.  **목표가 (Take Profit):** 진입가 대비 **+10%** 도달 시 즉시 익절.
2.  **손절가 (Stop Loss):** 진입가 대비 **-4%** 도달 시 즉시 손절 (고정 손절).
3.  **장마감 청산 (EOD):** 오후 15:20분까지 청산되지 않은 모든 포지션은 시장가로 전액 매도하여 당일 청산을 원칙으로 합니다. (오버나이트 배제)
